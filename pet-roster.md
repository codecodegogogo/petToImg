# 宠物正文配图角色库

本文件夹是 Ian 风格中文正文配图的固定宠物主角来源。动物名字使用图片文件名，不要另起名字。

## 图片文件

- `小黄.png`：小黄
- `小狗儿.png`：小狗儿
- `蒂娜.png`：蒂娜

## 小黄

- 雌性，年龄2岁
- 浅黄大狗，体型修长，腿长，耳朵直立偏大。
- 尾巴常向上卷起，姿态机警、行动感强。
- 适合动作：巡逻、拉路径、开路、守门、快速搬运、发现断点、把东西叼走。
- 性格写法：好奇、认真、警觉、像负责外场执行的人。
- 可用短对话：`我先探路` / `这边断了` / `跟上` / `别堵门`

## 小狗儿

- 雄性，年龄9岁
- 橙黄小狗，体型更小更圆，白色胸口和白爪明显。
- 一只耳朵常立起，另一只耳朵半垂，表情有点无辜。
- 适合动作：试探、蹲守、被卡住、举手提问、拖小物件、钻进入口、看错方向。
- 性格写法：好奇、热心、略笨拙但认真、聪明。
- 可用短对话：`我试试` / `等等我` / `这是入口吗` / `先别慌`

## 蒂娜

-雌性，年龄1岁
- 黑猫，黄眼睛，尖耳朵，身体线条细长，尾巴灵活。
- 常见姿态有端坐、伏低、抬爪、警惕行走、蜷成一团。
- 适合动作：观察、判断、按下开关、踩住关键点、钻进暗处、监督、冷静记录。
- 性格写法：活泼、挑剔、像系统里的审稿人或守门员。
- 可用短对话：`不对` / `先看证据` / `这里有洞` / `我按住了`

## 随机组合规则

每张正文配图从三只宠物里选择 1-3 只出场：

- 单只：适合一个明确动作、一个判断、一个状态。
- 双只：适合对比、协作、拉扯、输入输出、角色分工。
- 三只：适合系统分流、闭环、多阶段、共同搬运或集体误入一个结构。

随机组合要服从正文表达。先判断结构需要几只，再轮换选择宠物；连续多张图尽量让三只都有出现机会。

## 对话规则

可以根据图片内容、宠物性格和当前正文隐喻添加少量对话。对话不是必需项，只在能增强画面时使用。

- 每张图最多 1-3 个小气泡。
- 每个气泡 2-8 个中文字符。
- 对话要服务结构表达，不要变成段落解释。
- 对话可以暴露角色性格：小黄负责推进，小狗儿负责试探，蒂娜负责判断。
- 不要写大段台词、网络梗、拟人剧情或可爱卖萌口癖。
- 对话气泡也算中文标注，总文字量仍需克制。

## 风格约束

- 16:9 横版中文正文配图。
- 纯白背景，大量留白。
- 黑色手绘线稿为主，少量红色、橙色、蓝色中文手写批注。
- 宠物必须承担核心动作，不能只是站在旁边。
- 保留每只宠物的 2-4 个可识别特征。
- 不要画成真实照片、萌宠海报、贴纸、儿童卡通或商业插画。

## 样貌一致硬规则

三张 PNG 是权威参考。生成 Ian 风格配图时，必须先查看图片，再把宠物转译成手绘角色。不要只按文字描述生成类似动物。

- 小黄必须保持浅黄大狗、修长体型、直立大耳、上卷尾巴。
- 小狗儿必须保持橙黄小狗、白胸白爪、一耳立一耳半垂、圆一些的体型。
- 蒂娜必须保持黑猫、黄眼睛、尖耳朵、细长身体和灵活尾巴。
- 不允许把它们画成通用猫狗、儿童卡通、贴纸、毛绒玩具或新的虚构角色。
- 如果生图工具不能附带参考图，只能生成草稿，不能声称样貌严格一致。
