# QA Checklist

## 必过项

- 是 16:9 横版。
- 背景是干净白底。
- 有宠物主角：小黄、小狗儿、蒂娜中的 1-3 只。
- 宠物主角承担核心动作，不只是装饰。
- 没有复刻旧案例构图，而是为当前文章生成了新隐喻。
- 画面怪诞、有创意、有意思。
- 简洁清爽，主体不超过画面约 60%。
- 一张图只讲一个核心结构。
- 中文标注少、短、能读。
- 橙色只用于主路径或箭头。
- 红色只用于重点、问题、提醒或结果。
- 蓝色只用于补充说明、反馈或系统状态。

## 样貌一致硬规则

- 必须和根目录参考 PNG 的动物样貌一致。
- 小黄、小狗儿、蒂娜不能被画成通用猫狗或新角色。
- 体型、毛色、耳朵、尾巴、脸型、眼睛和典型姿态如果明显偏离参考图，直接失败。
- 如果生成过程没有使用参考图作为视觉输入，只能作为草稿，不能标记为最终合格图。

## 宠物主角加项

- 宠物至少保留 2-4 个可识别特征，例如体态、毛色、花纹、耳朵、尾巴、眼神或姿势。
- 多只宠物同屏时能区分，不要变成通用猫狗。
- 宠物没有变成照片写实、毛绒玩具、贴纸、儿童卡通或萌宠海报。
- 宠物正在做核心动作；如果拿掉宠物后图仍完全成立，说明不合格。
- 连续多张图的宠物组合有轮换，不要总是同一只出场，除非用户指定。
- 对话气泡最多 1-3 个，每个 2-8 个中文字符，并且服务当前动作。

## 失败信号

出现以下情况，重生成或局部编辑：

- 左上角有“常见坑 / Workflow / 系统架构图 / 路线图”等标题。
- 宠物像通用萌宠、贴纸头像、宠物写真或商业海报。
- 对话太多、太长、像剧情台词，或盖过正文标注。
- 画面像 PPT、课程课件、正式流程图。
- 元素太多、箭头太多、节点太多。
- 文字变成大段解释。
- 背景有纸纹、阴影、渐变、米色、噪点。
- 真实 UI 截图或科技感界面。
- 中文错字严重或标注不可读。
- 画面太死板，没有荒诞隐喻。
- 和旧案例构图过于相似，没有为当前文章重新发明隐喻。

## 迭代方法

- 太普通：让宠物主角成为动作主体，加入一个奇怪但成立的隐喻。
- 太复杂：删节点，只保留一个动作和 3-5 个短标注。
- 太可爱：强调 deadpan、not cute、not mascot、not pet poster。
- 太 PPT：去掉标题、边框、整齐网格和过多箭头，改成手绘场景。
- 太像旧案例：保留核心意思，换掉主物件和主角动作。
- 宠物不可识别：减少场景复杂度，强化每只宠物的轮廓、花纹、耳朵、尾巴或标志姿势。
- 文字错：优先局部编辑；错得多就重生成并减少标注数量。

## 交付判断

高质量图应该让读者先觉得“有点怪”，然后 1 秒内看懂结构。

如果第一眼像教程页、宠物海报或萌宠表情包，而不是白纸上的怪诞产品草图，就不合格。
